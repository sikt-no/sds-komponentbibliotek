# Sikt Design System

Always read `references/guides/rules.md` before writing any code.

## Where to look

1. **Core rules** (mandatory patterns, common errors, verify checklist) → `references/guides/rules.md` — read this first
2. **Component API** → `references/components/<slug>.md` — start with `references/index.md` for the full list
3. **Tokens** → `references/tokens/colors.md`, `spacing.md`, `typography.md`, `effects.md`, `base.md`
4. **Icons** → `references/icons.md`
5. **Which component to use?** → `references/guides/component-decision-tree.md`
6. **Forms / validation** → `references/guides/forms.md`
7. **CSS setup, tokens, page shell** → `references/guides/styling.md`
8. **Next.js** (CSS imports, App Router, `'use client'`, fonts) → `references/guides/nextjs.md` — read if the project uses Next.js
9. **Brand fonts & Content-Security-Policy** → `references/guides/fonts-and-csp.md`
10. **Responsive design** (breakpoints, mobile-first, layout patterns) → `references/guides/responsive.md`
11. **Typography** (Heading vs visual size, variants) → `references/guides/typography.md`
12. **UX patterns** (when/where to use components) → `references/guides/patterns/` — `header.md`, `dialog.md`
13. **Custom components** (when SDS has no match) → `references/guides/custom-components.md`
