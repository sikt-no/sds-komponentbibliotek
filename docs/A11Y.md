# Accessibility

All components should be accessible!  
More on [accessibility in Sikt](https://isikt.sharepoint.com/sites/Innsikt-OmOss/SitePages/Universell-utforming.aspx).

## Testing

Code is automatically checked with [eslint-plugin-jsx-a11y](https://github.com/jsx-eslint/eslint-plugin-jsx-a11y).  
All components should be tested with [jest-axe](https://github.com/nickcolley/jest-axe) for accessibility issues.  
**Note** These tests only catches a small amount of issues and manual testing is still required by both designers & developers.  
**Note** Even though the components have unit tests you still need to write integration tests in your own application where you have used them.

## Relative sizes

Relative size tokens (`--sds-size-text-<size>`) should be used on user font size setting scalable properties like `font-size`, `line-height`, etc for accessibility purposes.

## Motion

The Core package CSS reset includes a media query for reduced motion that disable animations and transformations.

## Elements

### `<abbr>`

The [abbreviation element is not accessible by itself](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/abbr#accessibility_concerns). It can be hard to understand without correct text content and can't be accessed by keyboard.

1. Spell out the acronym or abbreviation in full the first time it is used on a page.
2. Make sure to use accessible markup. Tabindex makes is accessible by keyboard and class name will display the content.

```html
<abbr class="sds-typography-abbr" tabindex="0" title="Sikt Design System">
  SDS
</abbr>
```

There are an `<Abbrevation />` component in the core package that solves this for you.

### `<input type="date">`

The input element type date is not accessible and building one that is can be hard. Give the user an alternative to selecting a date in a calendar, like typing into input fields.

### Maps

Interactive maps such as Google Maps are not accessible. However these are not required to be as long as the information is conveyed in an alternative form, like an address, for more information see [UUTilsynet](https://www.uutilsynet.no/regelverk/kva-seier-forskrifta/153#unntak_for_kart_og_karttenester_).

## Stylesheets

### CSS content

If CSS content property carry meaning it need an accessible alternative [since it is not accessible by itself](https://developer.mozilla.org/en-US/docs/Web/CSS/content#accessibility_concerns). CSS alternatives are not fully supported yet, see [CSS property: content: Alternative text](https://caniuse.com/mdn-css_properties_content_alt_text) or [CSS property: alt](https://caniuse.com/mdn-css_properties_alt). So for the time being valid options are using `aria-label` or a `.sds-screen-reader-only` element containing the additional information.
